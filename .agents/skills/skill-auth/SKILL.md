# Skill Auth

这是“用户与权限管理”项目的公开交接 Skill，供进阶同事在 Codex 中基于本仓库资料进行复杂问答。

## 核心能力

本 Skill 只处理以下三类问题：

1. 查询流程和页面索引。
2. 回答已有权限逻辑。
3. 组合多条已确认规则，回答比较或关联问题。

## 资料范围

- 权限逻辑只使用 `references/逻辑规则.md`。
- 流程、页面和 Pixso 地址只使用 `references/页面索引.md`。
- `references` 中只包含当前公开网站允许展示的交接资料。
- 不得引用私有 `workskills`、个人资料、历史对话或未确认方案。

## 回答要求

回答必须包含：

- 结论
- 必要说明
- RULE 或 PAGE 来源

查询页面时，还应给出相关 Pixso 地址。

如果没有相关规则或页面记录，固定回答：

现有交接资料中没有相关记录，无法根据已有规则判断。

不得根据行业常识、相似功能、页面漏改文案或个人推断补充项目结论。

## 当前特别注意

- 用户组本身不支持限制，只支持授予和回收。
- “在具体用户上限制其从用户组继承来的权限”属于待确认的二次授权相关内容，不得作为确定结论。
- 角色只有授予和回收，没有限制。
- 稿件中的“岗位”统一理解为角色下的动态模板。
- 待确认内容不得作为现行结论回答。
