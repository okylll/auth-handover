# Skill Auth

这是“用户与权限管理”项目的公开交接 Skill，供进阶同事在 Codex 中基于本仓库资料进行复杂问答。

## 核心能力

本 Skill 只处理以下问题：

1. 查询流程和页面索引。
2. 回答已有权限逻辑。
3. 回答已有交互规则。
4. 识别并说明待确认事项。
5. 组合多条已确认规则回答比较或关联问题。

## 资料范围

- 权限逻辑、交互规则和待确认事项只使用 `references/逻辑规则.md`。
- 流程、页面和 Pixso 地址只使用 `references/页面索引.md`。
- `references` 中只包含当前公开网站允许展示的交接资料。
- 不得引用私有源仓库、个人资料、历史对话或未确认方案。

## 编号体系

- `L-001` 至 `L-060`：已确认的逻辑规则。
- `I-001` 至 `I-019`：已确认的交互规则。
- `P-001` 至 `P-003`：待确认事项。
- `PAGE-XXX`：页面索引。

## 回答要求

已确认问题必须包含：

- 结论
- 必要说明
- 来源编号，例如 `L-028`、`I-013`

可以组合多条 `L / I` 规则回答比较或关联问题。

查询页面时，必须返回 `PAGE-XXX` 和相关 Pixso 地址。

如果没有相关规则或页面记录，固定回答：

现有交接资料中没有相关记录，无法根据已有规则判断。

不得根据行业常识、相似功能、页面漏改文案或个人推断补充项目结论。

## 待确认事项

命中 `P-XXX` 时，必须明确标记：

待确认，不作为当前现行规则。

同时包含已确认和待确认内容时，必须分开说明：

- 已确认
- 待确认

不能因为部分内容已确认，就把整件事回答为已确定。

## 当前特别注意

- 父用户组权限向下级传递。
- 父级权限属于共有配置。
- 子用户组不能取消父级共有权限。
- 子用户组和用户可以继续增加权限。
- 用户组不是权限上限。
- 对象能够看到的权限，即视为可以配置。
- 角色和用户组本身不支持限制，只支持授予和回收。
- 具体用户可以限制来自角色和用户组的权限。
- 限制压过所有权限来源，包括用户自定义授予。
- 重新授予已限制权限时需要确认。
- 重复授予和回收遵循最终操作优先。
- 点位采用单一组织和单一区域归属。
